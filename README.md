# MD Editor

마크다운 파일을 앱 내부 저장소로 가져와 폴더 트리로 정리하고, 읽기 좋게 렌더링하며,
직접 편집까지 할 수 있는 **브라우저 기반 마크다운 뷰어/에디터**입니다.

모든 데이터는 브라우저 **IndexedDB**에 저장되어 서버가 필요 없고, 새로고침해도 유지됩니다.

## 기술 스택

| 영역 | 사용 기술 |
|------|-----------|
| 프레임워크 | Next.js 16 (App Router) + React 19 + TypeScript |
| 빌드/배포 | 정적 익스포트 (`output: 'export'`) — 서버리스 |
| 내부 저장소 | IndexedDB (Dexie.js) |
| 상태 관리 | Zustand |
| 마크다운 렌더 | markdown-it + markdown-it-task-lists |
| 코드 하이라이트 | highlight.js |
| XSS 방어 | DOMPurify |
| 에디터 | CodeMirror 6 (+ one-dark 테마) |

> 클라이언트 전용(IndexedDB/File API) 앱이므로 진입 컴포넌트는 `next/dynamic`의
> `ssr:false`로 로드해 브라우저에서만 실행됩니다.

## 기능

### 📁 파일/폴더 관리
- `.md` 파일 **가져오기(import)** — 버튼 선택 또는 사이드바 **드래그앤드롭**(다중 지원)
- **새 파일 / 새 폴더** 생성 (새 파일은 바로 편집 모드로 진입)
- **이름 변경 / 삭제** (폴더는 하위 항목까지 재귀 삭제)
- 사이드바 **트리 탐색** (폴더 우선·이름순 정렬)
- **다중 선택** — `Ctrl/⌘+클릭`(토글), `Shift+클릭`(범위)로 여러 항목 선택
- **폴더 드래그 이동** — 파일·폴더를 끌어 다른 폴더로 이동, 빈 영역에 놓으면 루트로 이동
  - **여러 개를 한 번에 이동** (선택된 항목 전체를 함께 이동)
  - 상위·하위를 동시에 선택하면 하위는 상위와 함께 이동 (중복 이동 방지)
  - 자기 자신·하위 폴더로의 이동은 순환 방지로 차단

### 👁 뷰어
- 전체 마크다운 렌더링: 제목/리스트/표/코드블록+문법 하이라이트/링크/이미지/인용/체크박스/GFM
- **KaTeX 수식**(`$...$`, `$$...$$`) + **Mermaid 다이어그램**(```mermaid) 렌더
- 파일을 열면 항상 **문서 최상단부터** 표시
- 안전한 렌더 파이프라인: `markdown-it → DOMPurify sanitize`
- **목차(TOC)/아웃라인** — 제목 기반 패널, 클릭 시 해당 위치로 스크롤

### 🔗 문서 연결
- **위키링크** `[[문서]]`/`[[문서|표시이름]]` — 클릭 시 해당 파일로 이동(없는 문서는 빨강 표시)
- **백링크** — 이 문서를 참조하는 다른 문서 목록
- **그래프 뷰** — 위키링크 관계망을 SVG 그래프로 시각화(노드 클릭 시 열기)
- **태그** `#태그` — 사이드바에서 태그별 필터
- **즐겨찾기**(★) · **최근 연 파일**(빠른 열기에서 우선 표시)

### 🗂 생산성 · 기타
- **템플릿 / 데일리 노트** — 회의록 등 양식으로 새 문서, 오늘 날짜 노트
- **다국어 UI** — 한국어 / English / 日本語 전환 (`⋯` 메뉴)

### 📤 내보내기 · 백업
- **문서 내보내기** — `.md` 다운로드 / **HTML 다운로드**(이미지 인라인) / **인쇄·PDF**
- **백업/복원** — 전체 저장소(폴더·파일·이미지)를 JSON 파일로 내보내고 복원
  - 브라우저 데이터가 지워져도 백업으로 복구 가능

### ⚡ 빠른 이동
- **탭** — 여러 문서를 동시에 열고 전환. 탭마다 편집 상태(보기/편집·미저장 내용)를 개별 유지. **드래그로 순서 변경**
- **세션 복원** — 새로고침·재접속 시 열려 있던 **탭·활성 탭·보기/편집 모드**를 그대로 복원.
  자동 저장을 꺼둔 **미저장 편집 내용(draft)** 도 함께 복원됨
- **명령 팔레트** (`Ctrl/⌘+P`) — 파일 이름으로 즉시 열기 (최근 연 파일 우선, 키보드 탐색)
- **사용자 지정 단축키** — `⋯` 메뉴 → 단축키 설정에서 변경(중복 시 경고), 기본값 복원
- **도움말** — `⋯` 메뉴에서 사용법 + 마크다운 작성법 + 단축키 표 안내

### ✏️ 에디터
- **CodeMirror 6** 기반, 마크다운 문법 하이라이트
- **서식 툴바**(굵게/기울임/제목/링크/목록/인용/코드) + 단축키(`Ctrl+B/I/K`)
- **찾기·바꾸기**(에디터 내 `Ctrl+F` 검색/치환 패널)
- **보기 / 편집** 토글, 편집 모드는 **에디터 + 라이브 프리뷰** 좌우 분할(스크롤 동기화)
- **문서 통계** — 단어·글자 수, 예상 읽기 시간 상태바
- **버전 히스토리** — 저장 시점별 스냅샷 기록 + 이전 버전으로 되돌리기
- **자동 저장** (입력 멈춘 뒤 저장) — 체크박스로 켜고 끄기, 설정은 유지됨
  - 자동 저장 OFF 시 수동 저장 버튼 + `Ctrl/Cmd+S`

### 🖼 로컬 이미지 첨부 (편집 모드)
- **이미지 버튼 / 붙여넣기(Ctrl+V) / 드래그앤드롭** 으로 이미지 첨부
- 이미지는 앱 내부 저장소(blob)에 저장, 문서엔 `![이름](asset:<id>)` 로 참조
- 렌더 시 objectURL로 해석해 표시

### 🔍 검색
- **전체 파일 검색** (사이드바): 파일 이름 + 내용 전체 검색, 매칭 스니펫·강조 표시
- **문서 내 찾기** (`Ctrl/Cmd+F`): 열린 문서에서 찾기, 매칭 강조 + 현재 항목 이동(`Enter`/`Shift+Enter`)
- 두 검색 모두 **대소문자 구분(Aa)** 토글 지원
- 편집 모드의 `Ctrl+F`는 CodeMirror 자체 검색 사용

### 🎨 UI/테마
- **다크 / 라이트 모드** 토글 — 시스템 설정을 기본값으로 따르고, 선택은 유지됨 (렌더 깜빡임 없음)
- 뷰어·에디터·코드 하이라이트 모두 테마 연동
- **사이드바** 폭 드래그 조절, 숨김/열기 버튼, 창이 좁아지면(< 700px) 자동 숨김(오버레이 모드)

   │  ├─ library/             # 사이드바 본문(트리/검색/즐겨찾기/태그) + 즐겨찾기·최근·태그

### 저장소 스키마

**IndexedDB (Dexie)** — 문서 데이터 (영구)

```
nodes     { id, parentId, type:'folder'|'file', name, order, createdAt, updatedAt }  # 인접 리스트 트리
files     { nodeId, content }                                                        # 원본 마크다운
assets    { id, nodeId, blob, mime }                                                 # 첨부 이미지
snapshots { id, nodeId, content, savedAt }                                           # 버전 히스토리(파일당 최근 50개)
```

**localStorage** — 설정 · 세션 (키 prefix `md-viewer:`)

| 키 | 내용 |
|---|---|
| `theme` | 다크/라이트 (`dark` \| `light`) |
| `locale` | 언어 (`ko` \| `en` \| `ja`) |
| `sidebarWidth` / `sidebarCollapsed` | 사이드바 폭 / 접힘 |
| `autoSave` | 자동 저장 on/off |
| `favorites` | 즐겨찾기 파일 id 배열 |
| `recent` | 최근 연 파일 id 배열 (최신순, 최대 20) |
| `keybindings` | 사용자 지정 단축키 (액션 → 콤보) |
| `session` | **세션**: 열린 탭(`fileId`·`mode`·미저장 `draft`) + 활성 탭 |

> 문서·이미지는 IndexedDB, 설정과 세션은 localStorage에 나뉘어 저장됩니다.
> 백업 내보내기(JSON)는 IndexedDB 데이터(nodes·files·assets)를 담습니다.

## 설계 메모

- **`StorageAdapter` 추상화**: 모든 저장소 접근이 이 인터페이스를 통과합니다.
  나중에 **Tauri 데스크톱 앱**으로 확장할 때 `TauriFsAdapter`(실제 파일시스템)만 새로
  구현하고 `storage/index.ts` 한 줄만 교체하면 UI·렌더러·상태 코드는 그대로 재사용됩니다.
  인접 리스트 트리 모델이 실제 폴더 계층과 1:1 대응돼 이관이 단순합니다.
- **`renderer.ts`는 순수 함수**: 뷰어와 에디터 라이브 프리뷰가 동일한 렌더 파이프라인을 공유합니다.
